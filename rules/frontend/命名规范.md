# 前端命名规范

## 组件命名
- 使用 PascalCase 命名组件文件和导出名（`ProjectCard.tsx` → `export function ProjectCard()`）
- 文件名与组件名保持一致
- 复合组件放在同名子目录中（`pyramid/PyramidNavigation.tsx`、`pipeline/RuleCheckStep.tsx`）
- 使用 barrel exports（`index.ts`）统一导出

## Props 接口命名
- Props 接口使用 `{ComponentName}Props` 格式（如 `StageIndicatorProps`、`PRDPreviewProps`）
- 在组件文件内定义，使用 `export interface`
- 若 Props 简单（2-3 个属性），可用内联解构

## Hooks 命名
- 统一使用 `use` 前缀 + camelCase（`useCliChat`、`useAgentStatus`、`useFileWatcher`）
- 文件名与 hook 名一致（`useCliChat.ts`）
- 通过 `hooks/index.ts` 统一导出

## Context 命名
- Context 使用 PascalCase（`ProjectContext`）
- Provider 使用 `{Name}Provider`（`ProjectProvider`）
- 消费 hook 使用 `use{Name}`（`useProject`）

## 变量命名
- 变量和函数使用 camelCase（`currentLevel`、`handleSubmitAnswers`）
- 常量使用 UPPER_CASE（`PROJECT_ROOT`、`STATUS_FILE`）
- 布尔变量使用 `is/has/should` 前缀（`isLoading`、`hasRestoredStateRef`、`saveSuccess`）

## TypeScript 类型
- 对象形状使用 `interface`（首选）
- 联合类型使用 `type`（如 `type PipelineMode = 'quick' | 'feature' | 'full' | 'transform'`）
- 类型守卫使用 `is{TypeName}` 命名（如 `isAskUserQuestionInput()`）
- 类型导出使用 `export type { ... }` 与值导出分离

## Barrel Exports 规范
```typescript
// components/index.ts
export { Header } from "./Header";
export type { HeaderProps } from "./Header";
export { ProjectCard } from "./ProjectCard";
```
- 每个目录的 `index.ts` 导出该目录的所有公共组件和类型
- 子目录（如 `pyramid/`、`pipeline/`）各自维护 barrel
